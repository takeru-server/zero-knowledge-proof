ゼロ知識証明

ゼロ知識証明（Zero-Knowledge Proof, ZKP）は、ある主張が真であることを、相手にその主張の根拠を知らせずに証明する方法である。
これは、暗号学やデジタルセキュリティにおいて非常に重要な概念で、特定の情報の秘密を守りながら、その情報に関連する証明を行うことができる。

基本概念

ゼロ知識証明は、以下の三つの主要な特性を持っている：

  1. 完全性: 真であることを確認する側（検証者）は、証明する側（証明者）の持っている命題が真であるならば、真であることが必ず検証して正しさを確認できる。 …①
  2. 健全性: 証明者の持つ命題が偽であるなら、検証者は高い確率でそれが偽であると見抜ける。 …②
  3. ゼロ知識性: あらゆる場合において、検証者が証明者から何らかの知識（情報）を盗もうとしても、証明者の主張が真であること以外の情報は得られない。 …③

例

ゼロ知識証明を理解するための直感的な例として、洞窟の例（別名：アリババの洞窟）がよく使われる：

  1. 証明者（Peggy）は洞窟の中にある秘密の扉を開ける方法を知っていると主張する。
  2. 検証者（Victor）はその主張が真であることを確認したいが、実際に扉を開ける方法を知りたくないとする。
  3. 洞窟は二つの道に分かれており、Victorは洞窟の入り口で待ちる。
  4. Peggyは洞窟に入り、扉の向こう側に移動する。
  5. VictorはPeggyにどちらの道から出てきてほしいかをランダムに指示する。
  6. Peggyが正しい道を知っている場合、必ず指定された道から出てくることができる。 これを何度も繰り返すことで、VictorはPeggyが扉の秘密を知っていると確信できるが、実際の方法は知らないまま。

実際の応用

ゼロ知識証明は多くの実用的なアプリケーションに使用されている。例えば：

  • デジタル署名: 署名の検証ができるが、署名の秘密鍵が漏洩しないようにする。
  • 認証システム: ユーザーがパスワードを共有せずにシステムにログインできるようにする。
  • プライバシー強化技術: ブロックチェーン技術などで、トランザクションの正当性を証明しつつ、取引内容を秘密にする。

ゼロ知識証明は、プライバシーを保護しながら信頼性を確保するための強力なツール。この技術は暗号通貨やブロックチェーン技術においても注目を集めている。




証明について

これを数学的に証明するためには、プロトコルを設計し、そのプロトコルが以下の特性を満たすことを示す必要がある。
①, ②, ③の特性を満たすプロトコルの例として、以下に「離散対数問題」を用いたゼロ知識証明のプロトコルを示す。 ### 離散対数問題に基づくゼロ知識証明

前提条件

  • p は大きな素数である。
  • g は p を法とする原始根（すべての1以上p-1以下の整数が、ある k を使って gk mod p として表される）である。
  • y = gx mod p という形で証明者（Peggy）が秘密の値 x を知っていることを証明する。

プロトコルの手順

  1. コミットメント:
    • Peggy はランダムな数 r を選び、コミットメント t = gr mod p を計算して、検証者（Victor）に送る。
  2. チャレンジ:
    • Victor はランダムなビット c（0または1）を選び、Peggy に送る。
  3. レスポンス:
    • c = 0 の場合: Peggy は r を送る。
    • c = 1 の場合: Peggy は s = r + x mod (p-1) を計算し、送る。
  4. 検証:
    • c = 0 の場合: Victor は gr mod p が受け取った t と一致することを確認する。
    • c = 1 の場合: Victor は gs mod p が t · y mod p と一致することを確認する。

証明

完全性（Completeness）

  • 主張が真であり、Peggy が正直である場合、全ての手順が正しく進む。
  • c = 0 の場合、Peggy は正しい r を持っているため、t = gr mod p は一致する。
  • c = 1 の場合、Peggy の s = r + x mod (p-1) から gs = gr + x = gr · gx mod p = t · y mod p となり、検証に成功する。

健全性（Soundness）

  • 主張が偽であり、Peggy が正しい x を知らない場合、Peggy が r や s を適当に選んでも、検証者を騙せる確率は1/2である。複数回の試行でこの確率は急速に低下する。

ゼロ知識性（Zero-Knowledge）

  • Victor は r または s のいずれかを知るだけであり、Peggy の秘密 x については何も知らない。
    Peggy のレスポンスはランダムな挑戦 c に依存するため、Victor は追加の情報を得ることができない。
